--------------------------------------------------------------------------------
  1. EXECUTIVE SUMMARY
--------------------------------------------------------------------------------

Suraksha AI is an AI-powered hospital security intelligence platform designed
to protect healthcare professionals, monitor staff compliance, ensure patient
safety, and enforce hospital security policies 24/7 — using your existing CCTV
infrastructure with no new hardware investment required.

The system combines the power of Google's Gemini Vision AI with specialized
free Hugging Face models to deliver fast, accurate, multi-threat detection
across every area of a hospital — from emergency rooms to pharmacies, from
ICUs to parking lots.

"Suraksha" (सुरक्षा) means "protection" or "security" in Hindi — a fitting
name for a system built to protect the people who protect us.

--------------------------------------------------------------------------------
  2. PROBLEM STATEMENT
--------------------------------------------------------------------------------

Indian and global hospitals face a growing security crisis:

  → Attacks on doctors and nurses are increasing every year
  → Staff negligence (sleeping on duty, patient abandonment) goes unnoticed
  → Missing infants from maternity wards — a critical safety failure
  → Fire hazards in hospital labs and kitchens remain undetected until too late
  → Unauthorized persons accessing pharmacies, ICUs, and OTs
  → Patient falls in unmonitored corridors cause serious injuries
  → Manual CCTV monitoring is ineffective — humans miss 95% of incidents

Traditional CCTV systems record but do NOT analyze. Suraksha AI transforms
passive recording into active, intelligent, real-time security.

--------------------------------------------------------------------------------
  3. WHAT SURAKSHA AI DETECTS
--------------------------------------------------------------------------------

The platform is designed to detect the following hospital-specific incidents:

  3.1 VIOLENCE & AGGRESSION AGAINST STAFF
      -----------------------------------------
      • Physical attacks on doctors and nurses
      • Verbal aggression and threatening behavior
      • Mob formation around staff areas
      • Altercations in waiting rooms and corridors
      Detection: Gemini Vision + Emotion AI (HF: dima806/facial_emotions)

  3.2 STAFF NEGLIGENCE & DUTY VIOLATIONS
      -----------------------------------------
      • Nurse/doctor sleeping on duty
      • Patient call lights being ignored
      • Staff abandoning patient areas
      • Hygiene protocol violations (not wearing PPE)
      • Unauthorized loitering in restricted zones
      Detection: Gemini Vision (hospital behavior analysis)

  3.3 MISSING INFANT / NEWBORN PROTECTION
      -----------------------------------------
      • Unauthorized removal of infants from maternity/NICU
      • Perimeter monitoring of all ward exits
      • Alert if an infant is carried near unauthorized exits
      • After-hours movement detection in newborn units
      Detection: Gemini Vision Perimeter + Person Tracking

  3.4 FIRE & SMOKE DETECTION
      -----------------------------------------
      • Real-time fire and smoke detection across all zones
      • Kitchen, lab, generator room monitoring
      • Automatic evacuation alert trigger
      • Multi-zone simultaneous monitoring
      Detection: HF Model → EdBianchi/vit-fire-detection (free, specialized)

  3.5 PATIENT FALLS
      -----------------------------------------
      • Fall detection in wards, corridors, washrooms
      • Unattended patient getting out of bed alerts
      • Night-time monitoring when staff presence is low
      • Immediate nursing station notification
      Detection: Gemini Vision (pose + scene analysis)

  3.6 UNAUTHORIZED ZONE ACCESS
      -----------------------------------------
      • ICU and OT perimeter enforcement
      • Pharmacy restricted area monitoring
      • Drug vault access tracking
      • After-hours intrusion detection
      • Visitor overstay and trespass alerts
      Detection: Gemini Vision + DETR Object Detection (HF)

  3.7 CROWD & OVERCROWDING
      -----------------------------------------
      • Emergency room overcrowding alerts
      • Crowd gathering analysis (mob risk detection)
      • Person counting for capacity enforcement
      Detection: HF Model → facebook/detr-resnet-50 (free)

  3.8 EMOTIONAL DISTRESS / AGGRESSION FLAGS
      -----------------------------------------
      • Facial expression analysis — anger, fear, distress
      • Early warning before physical violence occurs
      • De-escalation support through preemptive security alerts
      Detection: HF Model → dima806/facial_emotions_image_detection (free)

--------------------------------------------------------------------------------
  4. AI TECHNOLOGY STACK
--------------------------------------------------------------------------------

Suraksha AI uses a MULTI-MODEL approach to distribute AI workload:

  TASK                        MODEL                               COST
  ─────────────────────────────────────────────────────────────────────
  Fire & Smoke Detection      EdBianchi/vit-fire-detection         FREE (HF)
  Facial Emotion/Aggression   dima806/facial_emotions_image        FREE (HF)
  Person & Crowd Detection    facebook/detr-resnet-50              FREE (HF)
  Complex Scene Analysis      Google Gemini 1.5 Flash              Gemini API
  Violence / Staff Behavior   Gemini Vision (hospital prompt)      Gemini API
  Incident Summarization      Gemini 1.5 Flash                     Gemini API

  Design principle:
    Simple visual classification → Free Hugging Face models
    Complex human behavior analysis → Google Gemini Vision

  API KEYS NEEDED:
  ─────────────────────────────────────────────────────────────────────
  GEMINI_API_KEY     → Google AI Studio (free tier available)
                       Get at: https://aistudio.google.com

  HF_API_KEY         → Hugging Face Access Token (completely free!)
                       Get at: https://huggingface.co/settings/tokens
                       Sign up free → Settings → Access Tokens → New Token

  SUPABASE_URL       → Already configured (database + file storage)
  SUPABASE_ANON_KEY  → Already configured (authentication)

  5. PLATFORM FEATURES
--------------------------------------------------------------------------------

  5.1 LIVE DASHBOARD — Hospital Security Matrix
      → Real-time multi-camera grid view of all hospital zones
      → Color-coded incident severity across all feeds
      → Live Event Feed with hospital-specific incident classification
      → Camera click-to-expand with full-screen viewing
      → Hospital-wide status: cameras online, active incidents, staff count

  5.2 AI VIDEO ANALYZER (Pre-Analyzer + Deep Analysis)
      → Upload any CCTV clip for AI analysis
      → Multi-model parallel scan (fire, aggression, crowd simultaneously)
      → Gemini Vision scene analysis with incident timeline
      → Exportable evidence report with timestamps
      → Pre-analysis metadata scan before full AI processing

  5.3 REALTIME LIVE FEED
      → Real-time streaming camera analysis
      → Frame-by-frame AI inference on live footage
      → Instant alert dispatch via Telegram, Email, WhatsApp
      → Live incident log with camera source identification

  5.4 INCIDENT LIBRARY
      → All analyzed clips stored and searchable in Supabase
      → Incident timeline generated per video
      → Filter by department, incident type, date, severity
      → Evidence export for legal / HR / administrative action

  5.5 MULTI-CHANNEL ALERT SYSTEM
      → 📱 Telegram (hospital security group — instant notification)
      → 📧 Email (HOD, Chief Security Officer)
      → 💬 WhatsApp (duty nurse / on-call doctor)
      → Alert contains: incident type, hospital zone, timestamp, camera ID
      → One-click "Dispatch Security" from dashboard event feed

  5.6 SECURE AUTHENTICATION
      → Supabase-powered login/signup
      → Role-based access: Security Chief / HOD / Nurse / Admin
      → Session management and activity audit trail

  7. TECHNICAL ARCHITECTURE
--------------------------------------------------------------------------------

  Frontend:    Next.js 14 (App Router) + TypeScript + Tailwind CSS
  Backend:     Next.js API Routes (serverless edge functions)
  Database:    Supabase (PostgreSQL + File Storage)
  Auth:        Supabase Auth (JWT-based, role-ready)
  AI Layer:
    → Google Gemini 1.5 Flash (complex scene + behavior analysis)
    → Hugging Face Inference API (3 specialized free models)
    → Parallel model execution (minimizes API response latency)
  Alerts:
    → Telegram Bot API
    → Email via SMTP / Resend
    → WhatsApp Business API
  UI:          React + Framer Motion + Lucide Icons
  Routing:     Next.js App Router (file-based)

  Key API Routes:
    POST /api/hf-analyze     → Hugging Face multi-model inference
    POST /api/pre-analyze    → Video metadata pre-scan
    POST /api/analyze        → Full Gemini video analysis
    POST /api/send-telegram  → Telegram alert dispatch
    POST /api/send-email     → Email alert dispatch
    POST /api/send-whatsapp  → WhatsApp alert dispatch
    POST /api/chat           → AI chat interface (Gemini)
    POST /api/upload         → Video upload to Supabase storage

  Key Pages:
    /                  Landing page + feature overview
    /surveillance      Hospital Security Matrix (live dashboard)
    /pre-analyzer      Video upload + AI analysis flow
    /realtime          Live camera feed with real-time inference
    /library           Incident archive + past video analyses
    /upload            Direct video upload to storage
    /sign-in           Secure hospital staff login
    /sign-up           Hospital onboarding and registration

--------------------------------------------------------------------------------
  8. HOW IT WORKS — STEP BY STEP
--------------------------------------------------------------------------------

  Step 1: Hospital connects existing CCTV to Suraksha AI platform
  Step 2: All cameras appear in the live Hospital Security Matrix
  Step 3: System continuously runs parallel AI inference:
            Fire/smoke   → HF model check every 5 seconds
            Crowd/persons → HF model check every 10 seconds
            Behavior     → Gemini Vision on flagged/suspicious frames
  Step 4: When an incident is detected:
            → Alert appears instantly in real-time Event Feed
            → Security officer receives Telegram + Email + WhatsApp
            → Relevant camera is highlighted in dashboard
  Step 5: Security officer can:
            → Click event to expand that camera's live view
            → Issue a formal escalation alert to senior staff
            → Export incident report for HR / legal action
  Step 6: All footage is archived in the Incident Library

--------------------------------------------------------------------------------
  10. SETUP REQUIREMENTS
--------------------------------------------------------------------------------

  1. Clone the repository: kwala_ai-main
  2. Install dependencies:  npm install
  3. Create .env.local with:

       NEXT_PUBLIC_SUPABASE_URL=<your-supabase-url>
       NEXT_PUBLIC_SUPABASE_ANON_KEY=<your-supabase-anon-key>
       GEMINI_API_KEY=<your-gemini-api-key>
       HF_API_KEY=hf_xxxxxxxxxxxxxxxx          ← GET THIS FREE
       TELEGRAM_BOT_TOKEN=<telegram-bot-token>
       TELEGRAM_CHAT_ID=<hospital-security-group-id>
       WHATSAPP_API_KEY=<whatsapp-business-api-key>
       EMAIL_FROM=alerts@yourhospital.com
       RESEND_API_KEY=<resend-api-key>

  4. Run development server: npm run dev
  5. Open in browser: http://localhost:3000

  Notes:
  → HF_API_KEY is FREE — go to huggingface.co → sign up → Settings → Tokens
  → Only Supabase + Gemini keys are required for core functionality
  → Alert channels (Telegram, WhatsApp, Email) are optional but recommended
