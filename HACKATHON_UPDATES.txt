# SURAKSHA AI - Hackathon Updates & Performance Fixes

1. YOLO Model Upgraded to YOLO11
--------------------------------
- File: `python_backend/live_feed_server.py`
- Change: Upgraded the local inference model from `yolov8n.pt` to `yolo11n.pt`.
- Reason: The Hugging Face "YOLOv22" models are unofficial/unstable. YOLO11 is the latest, fastest, and most precise official model released by Ultralytics, giving you cutting-edge bounding box tracking.

2. Removed Live-Feed Freeze via Multi-Threading
--------------------------------
- File: `python_backend/live_feed_server.py`
- Change: Imported `threading` and wrapped the external Hugging Face API network calls inside a background worker thread (`threading.Thread`).
- Reason: Previously, when the system detected 2+ people interacting, it would send an API request to Hugging Face and pause the entire stream for ~1.5 seconds. The video feed will now remain buttery smooth since the API checks run in the background.

3. Enforced 60 FPS & 720p High-Resolution Capture
--------------------------------
- File: `python_backend/live_feed_server.py`
- Change: Added strict OpenCV initialization parameters:
    `cap.set(cv2.CAP_PROP_FPS, 60)`
    `cap.set(cv2.CAP_PROP_FRAME_WIDTH, 1280)`
    `cap.set(cv2.CAP_PROP_FRAME_HEIGHT, 720)`
- Reason: OpenCV defaults to 30 FPS. By hardcoding these properties, the camera hardware is forcefully opened at 60 FPS and 720p HD resolution for a premium, zero-latency appearance.

4. VS Code Virtual Environment Sync Issue Identified
--------------------------------
- Issue: "Cannot find module 'ultralytics'" IDE error.
- Fix Provided: Explained that the IDE was using the global Python configuration (`C:\\Python314`) instead of the active `venv` environment (`venv\\Scripts\\python.exe`). Taught how to fix this by using "Python: Select Interpreter" in VS Code to point to the `venv`.
